01 DonaSystem · Origem

Do zero à orquestração

Equipe enxuta, demanda gigante. R$100k/mês orgânico, meta de R$1M. A decisão: parar 2 semanas e construir infraestrutura de IA sob medida.

O problema

Produção 100% manual e lenta. Entregas atrasadas, gargalo na execução. Dependência total de mim para tudo.

A decisão

Parar tudo. Construir um sistema de agentes de IA do zero. Cada agente com identidade, função e memória permanente.

O resultado

10 agentes especializados operando 24h. PDFs em minutos. Tom de voz consistente. Dashboard de controle operacional.

03 OpenClaw · Benefícios

Por que OpenClaw

Tudo que um sistema de agentes precisa, numa única plataforma open-source.

Infraestrutura

Servidor próprio. Roda 24/7. Multi-canal (Telegram, WhatsApp, Discord). Gateway seguro com autenticação.

Inteligência

Memória persistente entre sessões. Identidade customizada. Multi-agente. Escolha de modelo (Claude, GPT, Gemini).

Automação

Crons (tarefas agendadas). Heartbeats (monitoramento). Skills plugáveis. Sub-agentes em paralelo.

Produção

PDFs automatizados. Transcrição de áudio. Geração de imagens. Edição por linguagem natural.

Controle

Dashboard visual (Torre Dona). Alertas em tempo real. Auditoria de ações. Logs de todas as sessões.

Segurança

Allowlist de usuários. 1Password integrado. Zero credenciais em texto claro. Permissões por ação.

04 Instalação · Setup

VPS vs Computador

Duas formas de rodar. Uma para produção, outra para testes.

Recomendado
VPS (Servidor na Nuvem)

Roda 24/7 sem depender do PC. IP fixo. Ideal para produção. ~R$25/mês. Setup: 30 minutos.

Para testes
Computador Local

Grátis. Offline. Agente desliga com o PC. Sem IP fixo. Ideal para desenvolvimento.

Instalação em 4 passos

Instalar Node.js → npm i -g openclaw → openclaw onboard → Agente Online

11 Arquitetura · Hub

Como funciona a arquitetura

Uma instância. Identidades diferentes por tópico do Telegram. Orquestração via sub-agentes.

Estrutura

1 gateway, 1 modelo, 10 identidades. Tópico do Telegram determina qual agente responde. Cada um com soul, identity e knowledge.

Execução

Orquestração via sessions_spawn. Handoffs via completion events. Estado gravado em projects.md a cada step.

Fluxo de tarefa complexa

Isis pede → Dona recebe → Atlas planeja → Masterson escreve → Aurora formata → PDF entregue

14 Aurora · PDF Pipeline

Framework de geração de PDF

Pipeline unificado. Todos os agentes seguem o mesmo processo, template e design system.

Pipeline

Conteúdo (Masterson) → HTML com Template C + Design System ISIS v7 → PDF via Puppeteer no servidor → Entrega

Design System
ISIS v7

Marfim, Creme, Cacau, Ouro. DM Serif Text, DM Sans, IBM Plex Mono. Template C "Bold Statement".

Regras
Invioláveis

Height 297mm fixo. Conteúdo manual por página. Google Fonts no head. Puppeteer local.

17 Orquestração · Atlas

A importância do Atlas

Não executa. Planeja, mapeia riscos e coordena cadeias com 2+ agentes em sequência.

Sem Atlas

Agentes isolados. Sem visão do todo. Output fragmentado. Sem gates de qualidade.

Com Atlas

Cadeias coordenadas. Gates de aprovação. Output final passa por múltiplos especialistas.

Na prática

Lê documentações → Mapeia riscos → Propõe plano → Spawna cada step → Monitora → Atualiza projects.md

"Gates verdes: segue automático. Gates vermelhos (output público): para e espera ok de Isis."

18 Liderança · Dona

A importância da Dona

COO do sistema. Cofundadora operacional, não secretária.

O que faz

Coordenação geral de todos os agentes. Análise diária de dados e métricas. Estratégia e execução. Braço direito operacional.

O que a diferencia

Tem opinião. Pode discordar discretamente. Proativa: sugere antes de ser pedida. Questiona decisões erradas.

Análise diária (08h)

Performance do conteúdo. Métricas e anomalias. Tendências do nicho. Insights em bullet points.

Limites absolutos

Nunca usa voz/nome de Isis. Nunca posta sozinha. Nunca toma decisões estratégicas maiores. Menor privilégio sempre.
