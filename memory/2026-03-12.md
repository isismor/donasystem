## Aprendizados Técnicos

### Device Identity no OpenClaw
- HTTP (non-secure context) bloqueia WebCrypto → browser não consegue gerar device identity
- `dangerouslyDisableDeviceAuth=true` NÃO remove totalmente a exigência
- O bypass só funciona quando: `isControlUi=true` E `sharedAuthOk=true` (token correto enviado)
- `isControlUi` depende de `client.id === "openclaw-control-ui"` (hardcoded)

### Variáveis NEXT_PUBLIC_*
- São injetadas no BUILD TIME do Next.js
- Alterar .env sem rebuild não tem efeito no browser
- Fluxo: editar .env → `next build` → `next start`

### Lógica de connect no código fonte
- Arquivo: `gateway-cli-vk3t7zJU.js`
- Função: `evaluateMissingDeviceIdentity()`
- Fluxo: hasDeviceIdentity? → trustedProxy? → isControlUi + allowBypass? → roleCanSkipDeviceIdentity(operator + sharedAuthOk)? → reject
